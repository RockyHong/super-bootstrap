# GAP-030 — missing alignment-brief checkpoint at route→implement handoff

## Findings

- root cause: **an absence, split across two real levers — diagnosis confirms both contribute, (a) is load-bearing.** The recurring symptom (gateway enters build with wrong framing, forcing manual re-triage after build spend) has two upstream causes, matching the card:
  - **Lever (a) — no gateway checkpoint at route→implement.** The current mechanism is `CLAUDE.md` § Development Workflow *"Route line — state, don't gate"*: it confirms the **cluster + route** ("how I'll process this"), never the **premise / problem-context / proposed-solution** ("what I understand the problem to be + what I'll build"). That framing goes unstated until build output surfaces it. This is the only point that catches **both** failure modes named in the card — gateway misreads the card directly, AND gateway absorbs a misframed triage verdict — and it covers **all** entry paths (fresh feature, direct cluster-3, refactor, triage-verdict pickup). Necessary backstop.
  - **Lever (b) — triage framing discipline.** `agents/triage.md:86-89` (`## Decision needed`) already requires *"framed as a decision, not 'what should I do'"* + a recommendation, but has no rubric guarding option correctness (mutual-exclusivity, premise-accuracy, no wrong-premise smuggling). This is the "misframed A/B/C options" path from the GAP-027 source. Upstream defense-in-depth only — it exists solely on the triage lane, so it cannot cover non-triage entries or the gateway's own card-misread; cannot be the sole fix.
- scope reach: four harness surfaces. Ambient `CLAUDE.md` § Development Workflow (route→implement transition, highest centrality — loads every session) + its shipped-skeleton mirror `plugins/super-bootstrap/skills/harness-bootstrap/assets/claude-md-skeleton.md:26-28` (repo-boundary sync closure: the "state, don't gate" section is mirrored, so a lever-(a) edit propagates there) for lever (a); `plugins/super-bootstrap/agents/triage.md` (+ `skills/triage/SKILL.md` absorption protocol) for lever (b).
- attempted: cold-traced both levers against the named surfaces; confirmed no existing alignment-brief / premise-confirm mechanism anywhere (grep across all `.md`); confirmed the skeleton mirror carries the section in closure; checked `docs/decisions.md` for a closed fork (none relevant — sole hit was an unrelated `/decision`-capture rejection). Stopped at verdict: three of the four auto-fix criteria fail, so no scope.md is warranted — (2) scope is not contained (4 cross-surface harness targets incl. a shipped skeleton, not one feature surface); (3) test strategy is not unit/e2e (markdown harness — verification is `audit-harness-edits` + `writing-skills` behavioral micro-test, both human-eyeball class); (4) the scaled-gate design is a UX/ceremony trade-off the card itself flags ("a confirm round-trip on every pickup risks replacing one cost with another").

## Decision needed

- **D1 — which lever(s) carry the fix?** recommendation: **both, with (a) primary/load-bearing and (b) secondary hardening.** Lever (a) is the backstop that catches every failure mode and every entry path; (b) alone leaves the gateway's own card-misread and all non-triage entries uncovered. Shipping (a) without (b) still closes the symptom for triage-lane cards (the gate re-catches a misframed verdict); shipping (b) without (a) does not. So (a) is non-optional; (b) reduces how often (a) must fire.
- **D2 — the scaled alignment-brief gate design (lever a), the genuine user-judgment fork.** The card fixes the two endpoints (known-shape repeat → one-line brief that *states*; fuzzy/high-blast → full premise/scenario/solution/concerns + *explicit OK*), consistent with "state, don't gate." Open for the user: (i) the **trigger predicate** dividing state-only from gate — shape-familiarity is the named axis (per § Sizing), but the threshold and its signal need authoring; (ii) whether the one-line brief is **truly non-blocking** (post-and-proceed) or a soft pause; (iii) placement — a new envelope step between route and implement vs. an extension of the existing "Route line" section. recommendation: extend the existing "Route line — state, don't gate" section (its philosophy already generalizes: state the *framing* one-liner alongside the route line for known-shape; escalate to a full brief + OK on the same fork triggers that already stop the route line — ambiguous cluster, decisions.md conflict, high blast radius), rather than a separate gate step that risks the ceremony bloat the card warns against.
- **Route for the fix:** this is a **first-of-shape harness design** (the mechanism does not exist), so cluster 2 → `brainstorming` (design the scaled gate) before implement, not a direct cluster-3/7 edit. The CLAUDE.md ↔ skeleton sync and the triage-agent rubric are `writing-skills`-RED-eligible behavior-shaping prose; `load-harness-principles` pre, `audit-harness-edits` post. Fix-shape is `design`.
