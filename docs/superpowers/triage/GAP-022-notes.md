# GAP-022 — commit-channel single-channel gate vs SDD implementer-commits contract

## Findings

- root cause: **verified, two-layer conflict** — `commit-channel.sh:26-29` gates raw `git commit` to `agent_type ∈ {commit, *:commit, main}`; every other subagent is denied and routed to `/super-bootstrap:commit`. SDD's `implementer-prompt.md:38` step 4 ("Commit your work") + SKILL.md:56 diagram ("Implementer subagent implements, tests, commits, self-reviews") make the implementer commit its own work. A dispatched SDD implementer runs under neither `commit` nor `main` → denied at its commit step. Card premise confirmed factually accurate.
- **card framing incomplete (load-bearing correction):** admitting SDD implementers to `commit-channel`'s allow-list (the card's option A) does NOT unblock them — `docsync-gate.sh:77` would then deny the same commit for a missing/stale doc-sync token (no `/super-bootstrap:commit` ran to write it). The two hooks *jointly* enforce one invariant: **doc-sync scan ran before every commit**, funneled through the `/super-bootstrap:commit` door. A one-line allow-list widen leaves the commit still blocked one hook later.
- **the real fork is the invariant, not the allow-list:** letting SDD implementers commit directly bypasses (or must relocate) the doc-sync-before-commit gate that both hooks defend. That gate is the sb envelope's non-negotiable doc-sync step. This is an architecture/boundary call, not a mechanical composition.
- **an existing composition already resolves this class:** both hooks exempt drain-managed worktrees (`commit-channel.sh:12-13` note; `docsync-gate.sh:58-65`, the "free-commit contract") — a linked worktree under `.claude/worktrees/` commits freely, doc-sync deferred to branch absorption/merge. SDD-implementers-in-a-drain-worktree is a pre-built path where implementers commit their own work AND the doc-sync invariant survives (enforced at merge, not per-commit). `docs/decisions.md` rows 20 + 36 confirm the drain free-commit contract is deliberate, worktree-scoped design.
- scope reach: `commit-channel.sh` + `docsync-gate.sh` (both FROZEN hook assets) + CLAUDE.md § Dispatch / SDD-consumption routing + the drain worktree contract. Cross-hook, cross-layer — not one feature surface.
- cost-complaint overlap: the card's headline harm ("dispatch strictly more expensive than inline for exact-content/prose tasks") is partly a *routing* concern (don't dispatch transcription-grade work at all), which overlaps GAP-020 (re-review ceremony on transcription-grade fixes) and DEBT-012/013/014 (small-change cost knobs). The commit-channel gate is not the sole cost driver — even with free commits, dispatch of an exact-content edit carries brief + round-trip + report overhead over an inline edit.
- attempted: cold-traced both hooks end-to-end + SDD 6.1.1 implementer/reviewer prompts + SKILL.md flow + decisions.md for prior forks. Stopped at the design fork below — no static read picks between preserving vs relocating the doc-sync invariant; that is a user/architecture judgment, and criterion 4 (no user judgment) + criterion 2 (scope contained to one surface) both fail → surface.

## Decision needed

**Should sb preserve the single doc-sync-gated commit channel, or admit an SDD-implementer commit path — and if the latter, via which composition?**

Three coherent resolutions, each a different invariant stance:

- **A — Preserve the channel; carve out SDD step 4 (recommended).** Document in CLAUDE.md's SDD-consumption routing that sb overrides SDD's implementer-commits step: implementer reports built + file list, gateway commits via `/super-bootstrap:commit` (doc-sync gate intact). This is exactly what this session did — the card frames it as "pure overhead," but it is the invariant working as designed. Attack the *real* cost (dispatch > inline for exact-content tasks) at the routing layer with GAP-020 / DEBT-012-014, not by breaching the commit gate. Cheapest, zero-invariant-risk, no FROZEN-hook edit.
  - recommendation: **A**, with a cross-link to the cost-knob cards — because the doc-sync-before-commit invariant is load-bearing across the whole envelope, and the drain path (C) already exists for when free implementer commits are genuinely wanted, so widening the gate globally buys little and risks silent doc-sync bypass.
- **B — Route SDD through drain worktrees.** Reuse the existing free-commit contract: SDD implementers commit inside `.claude/worktrees/`, doc-sync enforced at merge. Delegation benefit lands with the invariant intact — but couples SDD consumption to drain's worktree lifecycle (heavier orchestration; may not fit every SDD run).
- **C — Widen both gates for a named SDD-implementer agent_type AND relocate doc-sync.** Admit the implementer in both hooks and move the doc-sync guarantee to the SDD review/merge boundary. Full delegation, but this is a genuine invariant relocation touching two FROZEN hooks — the largest blast radius, and the one that most needs explicit user sign-off.
